## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → DB_PATH | DB_PATH is read from the DB_PATH environment variable; controls the SQLite file location |
| process → SQLite file | The backend process is the only writer; no user-supplied SQL or paths in this phase |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | DB_PATH env var | accept | Single-user local/Docker app; DB_PATH is operator-controlled, not user-supplied. No path-traversal surface from external input in this phase. |
| T-01-02 | Tampering | seed re-run on restart | mitigate | Use INSERT OR IGNORE (never INSERT OR REPLACE); test_cash_balance_preserved asserts cash is not reset across init_db() calls. |
| T-01-03 | Denial of Service | missing parent directory | mitigate | get_connection() runs DB_PATH.parent.mkdir(parents=True, exist_ok=True) before connecting, preventing OperationalError on fresh volumes. |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed this phase (RESEARCH.md Package Legitimacy Audit: "No new packages"). All deps already locked in uv.lock. |
</threat_model>
