<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → POST /api/portfolio/trade | Untrusted JSON body crosses here (ticker, side, quantity) |
| router → SQLite | Multi-table writes must be atomic |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Tampering | POST /api/portfolio/trade body | mitigate | Pydantic TradeRequest validators: quantity > 0, side in (buy,sell), ticker uppercased and len <= 10 → 422 before any DB access |
| T-02-04 | Tampering | execute_trade SQL | mitigate | Parameterized queries only; `with con:` atomic block ensures cash/positions/trades never partially commit |
| T-02-05 | Tampering | trade pricing | mitigate | Price sourced only from price_cache.get(ticker); client cannot supply a price; missing price → 503 |
| T-02-06 | Denial of Service | snapshot_recorder | accept | Single background task, 30s cadence, single-user dataset; unbounded row growth acceptable at this scale (Open Question 2) |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages (RESEARCH.md Package Legitimacy Audit: zero new packages) |
</threat_model>
