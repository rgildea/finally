## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user → /api/chat (consumed in Plan 02) | Untrusted user message text crosses into the LLM prompt |
| backend → OpenRouter/Cerebras | API key leaves process to a third-party inference provider |
| LLM → backend | LLM-generated structured JSON (trades, watchlist actions) is untrusted output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | get_llm_response parsing untrusted LLM JSON | mitigate | Parse with `ChatResponse.model_validate_json`; catch `ValidationError` and return a safe ChatResponse — never pass unvalidated content downstream (V5 Input Validation) |
| T-03-02 | Information Disclosure | OPENROUTER_API_KEY usage in llm.py | mitigate | Key read only from environment (loaded via load_dotenv); never logged, never echoed into ChatResponse, never returned to client |
| T-03-03 | Denial of Service | Unbounded chat history / token budget | mitigate | load_recent_history caps at limit (default 10); reasoning_effort="low" bounds token use |
| T-03-SC | Tampering | `uv add litellm` supply chain | mitigate | litellm verified in RESEARCH Package Legitimacy Audit (PyPI, 3+ yrs, slopcheck [OK]); pinned via uv.lock — disposition Approved, no [ASSUMED]/[SUS] packages so no blocking checkpoint required |
</threat_model>
