| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, ticker-row inner padding, inline badge padding |
| sm | 8px | Between label and value in data cells, button inner y-padding |
| md | 16px | Panel inner padding, form field spacing, card padding |
| lg | 24px | Gap between panels in the grid layout |
| xl | 32px | Header height inner padding |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch target minimum 44px height for trade bar Buy/Sell buttons (accessibility)
- Header fixed height: 48px
- Watchlist row height: 52px (accommodates sparkline + two text lines)
- Trade bar height: 56px (fixed, anchored above footer)

**Rules:**
- Prices and all numeric market data use JetBrains Mono (tabular numerals, monospace prevents layout shift on digit change)

- Minimum font size: 12px (Label role); nothing smaller

## Layout Contract

The SPA uses a fixed-height viewport layout (no vertical scroll on the outer shell). Panels scroll internally.

```
┌──────────────────────────────────────────────────┐  48px   Header
├──────────────┬───────────────────────────────────┤
│              │  Main Chart (selected ticker)      │
│  Watchlist   ├───────────────────────────────────┤  ~55vh  Content row
│  Panel       │  Portfolio Heatmap | P&L Chart     │
│  (fixed      ├───────────────────────────────────┤
│   280px)     │  Positions Table                   │
│              ├───────────────────────────────────┤
├──────────────┴───────────────────────────────────┤  56px   Trade Bar
└──────────────────────────────────────────────────┘
```

**Breakpoint:** Desktop-first. Layout collapses to single-column below 1024px (tablet fallback, not primary concern for Phase 4).

## Interaction Contract

### Price Flash Animation

```css
.price-flash-up {
  background-color: rgba(34, 197, 94, 0.20);
  transition: background-color 500ms ease-out;
}
.price-flash-down {
  background-color: rgba(239, 68, 68, 0.20);
  transition: background-color 500ms ease-out;
}
/* JS: add class on price change, remove after 500ms */
```

Implementation: on each SSE price event, compare new price to previous. If higher, add `.price-flash-up` to the price cell for 500ms then remove. If lower, `.price-flash-down`.

### Connection Status Dot

| State | Color | Label (tooltip) |
|-------|-------|-----------------|
| connected | `#22c55e` | Live |
| reconnecting | `#f59e0b` | Reconnecting... |
| disconnected | `#ef4444` | Disconnected |

EventSource `onopen` → connected, `onerror` → reconnecting, custom reconnect retry → disconnected after 3 failed attempts.

### Ticker Selection

Clicking a watchlist row sets the active ticker. Active row gets left border 3px `#ecad0a`. Main chart re-renders for the selected ticker. No page navigation, no URL change.

### Trade Execution

No confirmation dialog. Buy/Sell button click immediately calls `POST /api/portfolio/trade`. On success: positions table and header cash balance update in place. On error: inline error text in red `#ef4444` appears below the trade bar inputs for 4 seconds then fades.

## Charting Library

**Chosen:** TradingView Lightweight Charts (`lightweight-charts` npm package)

**Rationale:** Canvas-based (not SVG), designed for financial tick data, handles frequent price updates without layout thrash, smaller bundle than Recharts for this use case.

**Usage:**
- Main ticker chart: `createChart` with `LineData` time series — price history built from SSE events stored in component state
- P&L chart: `AreaSeries` with data from `GET /api/portfolio/history`
- Sparklines in watchlist: `createChart` with minimal config (no axes, no grid), 60×28px canvas per row
